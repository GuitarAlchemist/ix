The router classifies each user query into exactly ONE of these 16 in-scope intents (or declines as out-of-scope):

2. skill.scaleinfo — notes of a NAMED scale (e.g. "notes in A natural minor")

4. skill.interval — the interval between two notes, or naming an interval (e.g. "interval from C to G")
5. skill.chordsubstitution — substitute chords for a given chord (e.g. "tritone sub for G7")
6. skill.beginnerchords — easy/first chords for beginners
7. skill.progressionmood — TRANSFORM a progression to sound darker/brighter (modal interchange, borrowed chords) — NOT describing, but changing mood
8. skill.circleoffifths — the circle of fifths; key relationships around it
9. skill.practiceroutine — practice routine / practice plan suggestions
10. skill.genreessentials — essential chords or scales FOR A GENRE (blues, jazz, country...)
11. skill.whatcanyoudo — meta: what the assistant can do / its capabilities
12. skill.transpose — transpose a chord or progression by an interval / to another key
13. skill.commontones — common tones shared BETWEEN two chords
14. skill.diatonicchords — the diatonic chords IN A KEY (e.g. "chords in the key of C")
15. skill.keyidentification — identify the KEY of a given chord progression
16. skill.progressioncompletion — suggest the NEXT chord to continue/finish a progression

Plus OUT-OF-SCOPE (expectedIntentId = "__none__"): non-music queries (weather, code, math, general chit-chat, guitar GEAR/buying questions which the theory router should decline, etc.).
