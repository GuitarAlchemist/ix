WORKSPACE-WIDE FINDINGS (for context, not blocking R7)
------------------------------------------------------
The scan found 1 vulnerability + 4 warnings across the full workspace.
None of them is in the ix-autograd dep cone; all belong to OTHER
crates in the workspace that ix-autograd does not depend on.

(1) RUSTSEC-2026-0049  —  rustls-webpki 0.103.9
    Title: CRLs not considered authoritative by Distribution Point
    Severity: vulnerability
    Solution: upgrade to rustls-webpki >= 0.103.10
    Affected path: reqwest 0.12.28 → ix-io, ix-agent
    Triage: NON-CRITICAL — ix-autograd does not use rustls, reqwest,
    or TLS at all. This advisory matters for ix-io and ix-agent when
    they make HTTPS requests. Fix by running `cargo update -p
    rustls-webpki` at workspace level in a separate commit.

(2) RUSTSEC-2024-0436  —  paste 1.0.15
    Title: paste crate no longer maintained
    Severity: unmaintained (warning)
    Solution: none available upstream; use alternatives
    Affected path: paste → ratatui 0.29.0 → ix-dashboard;
                   also metal 0.33.0 → wgpu → ix-gpu
    Triage: NON-CRITICAL — pure metadata warning, not a CVE.
    paste is a proc-macro with no runtime code. No action for R7.

(3) RUSTSEC-2026-0002  —  lru 0.12.5
    Title: IterMut violates Stacked Borrows
    Severity: unsoundness (warning)
    Solution: upgrade when upstream ratatui bumps lru
    Affected path: lru → ratatui 0.29.0 → ix-dashboard
    Triage: NON-CRITICAL — ix-autograd does not use lru.
    ix-cache has its own LRU implementation that is unaffected.

(4, 5) RUSTSEC-2026-0097  —  rand 0.8.5 AND rand 0.9.2
    Title: Rand is unsound with a custom logger using rand::rng()
    Severity: unsoundness (warning)
    Solution: upstream fix pending
    Affected paths:
      rand 0.8 → zbus → atspi → (indirect, ix-io/agent)
      rand 0.9 → tungstenite → tokio-tungstenite → ix-io
    Triage: NON-CRITICAL — the unsoundness only triggers when a
    custom global logger calls rand::rng(). IX does not install a
    custom logger, so the code path is unreachable. ix-autograd
    itself does not depend on rand.
